# Weekly Agenda Template

**How a typical 90-minute class works**

This course is **portfolio-first**, **evidence-based**, and **mastery-focused**.

---

## Three Places for Your Work

| Place | Role |
|---|---|
| **Student Hub / GitHub Classroom** | Instructions and today’s assignment |
| **Your GitHub repository** | **Learning evidence** — code, commits, reflections |
| **Your Notion portfolio** (optional) | Public showcase of your best finished work |

```text
Instructions tell you what to do.
Your GitHub shows how you learned.
Notion shows your best work to the world.
```

---

## What You Do Every Class

Your teacher will share today’s assignment (GitHub Classroom README or in-class mission). Each **90-minute** class typically follows:

| Time | Block | What you do |
|---|---|---|
| 0–10 min | Entry check | Short questions before learning |
| 10–25 min | Individual learning | One required resource + notes |
| 25–40 min | Talk round | Discuss with a partner |
| 40–55 min | Check-in | Group review; teacher clarifies stuck parts |
| 55–75 min | Practice | Main mission task |
| 75–85 min | Exit check | What you finished; what is still confusing |
| 85–90 min | Submission | Submit evidence listed in the assignment |

---

## What Counts as Success

**Not enough:** watching a video, copying code, or running something once.

**Required:** visible **learning evidence** — see [Assignment_Submission_Guide.md](Assignment_Submission_Guide.md).

**Mastery** means you can:

1. **Explain** — in your own words  
2. **Rebuild** — without copying full solutions  
3. **Debug** — read errors and fix step by step  
4. **Modify** — adapt the pattern to a new task  
5. **Apply** — use the idea in a later project
