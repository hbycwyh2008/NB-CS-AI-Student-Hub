# GitHub Classroom Setup Checklist

**For teachers — setup instructions only (no private curriculum content)**

---

## Step 1: Confirm template repositories

Confirm these repositories exist and are set as GitHub Template Repositories:

- `hbycwyh2008/lesson-01-github-readme-template`
- `hbycwyh2008/lesson-02-markdown-portfolio-template`
- `hbycwyh2008/lesson-03-html-css-profile-page-template`

For each repository:

1. Open the repository on GitHub.
2. Go to **Settings**.
3. Confirm **Template repository** is enabled.

---

## Step 2: Create GitHub Classroom

Create a classroom for this course.

Suggested name:

`NB CS AI Pathway 2026`

Open: [GitHub Classroom](https://classroom.github.com/)

## Step 4: Copy assignment links

After creating each GitHub Classroom assignment, copy the invitation link.

Paste the links into:

**[Week_01_Agenda.md](Week_01_Agenda.md)**

Replace each `TODO` link.

Then commit and push the Student Hub update so students see the live links.

---

## Step 5: Test as a student

Before class, test the flow:

1. Open Student Hub.
2. Open [Week 1 Agenda](Week_01_Agenda.md).
3. Click Assignment 1 link.
4. Accept the GitHub Classroom assignment.
5. Confirm a student assignment repository is created.
6. Edit a file.
7. Commit changes.
8. Complete reflection.
9. Check that the evidence is visible on GitHub.

---

## Important Boundary

Students should access:

- Student Hub
- GitHub Classroom assignment repositories
- their own portfolio repositories

Students should **NOT** access:

- the private master curriculum repository
- teacher-only lesson plans
- answer keys
- private prompts
- internal planning notes
