# Manual Assignment Submission Guide

## Why We Use GitHub Template Repositories

We use template repositories so every student starts from the same safe starter files while still creating their own independent assignment repository.

You do **not** need GitHub Classroom. You create your own repo from each template using **Use this template** on GitHub.

---

## Student Workflow

1. Open the assignment template repository (link in [Week_01_Agenda.md](Week_01_Agenda.md)).
2. Click **Use this template**.
3. Create a new repository under your own GitHub account.
4. Complete the assignment.
5. Commit your work.
6. Submit your repository link to your teacher (form, LMS, or as instructed in class).

---

## If You Cannot Open the Template

If the template repository does not open, do not panic. Tell your teacher. Do not create a blank repository unless your teacher tells you to.

---

## What to Submit

For each assignment, submit:

- your name
- assignment number
- GitHub repository link
- screenshot if required
- completed reflection file

Your **GitHub repository link** is the main evidence — not screenshots alone.

---

## Naming Rules

Use clear repository names:

- `assignment-01-github-readme-yourname`
- `assignment-02-markdown-portfolio-yourname`
- `assignment-03-html-css-profile-yourname`

Replace `yourname` with your name or student ID as your teacher directs.

---

## Teacher Review

The teacher will check:

- required files
- commit history
- README or project output
- reflection
- evidence of independent understanding

---

## Important

Do not submit screenshots only. Your GitHub repository link is the main evidence.
