# Project Showcase Guide

**Computer Science and AI Pathway — Student Hub**

---

## Purpose

Showcase day is when you **demonstrate real work** — not slides about work you cannot explain.

---

## Before Showcase Day

- [ ] Project runs locally (or is deployed with a working URL)
- [ ] README explains setup and features
- [ ] You have practiced a **3–5 minute demo**
- [ ] You prepared answers for: *What was hardest?* *What would you improve?* *How did you use AI?*

---

## Demo Structure (Suggested)

| Time | Section |
|---|---|
| 0:00–0:30 | Intro — project name and problem it solves |
| 0:30–2:30 | Live demo — core features only |
| 2:30–3:30 | Technical highlight — one thing you built yourself |
| 3:30–5:00 | Reflection — learning, debugging, AI use, next steps |

---

## What Judges / Teachers Look For

| Criterion | Strong Evidence |
|---|---|
| **Functionality** | Core features work; you handle small errors calmly |
| **Understanding** | You explain code and design choices |
| **Documentation** | README and repo are clear |
| **Responsible AI** | Honest disclosure; you own the final code |
| **Growth** | You can describe what improved since earlier projects |

---

## Presentation Tips

1. **Demo the happy path first** — then mention edge cases if time allows.
2. **Do not read** your README word-for-word.
3. **Show GitHub** — commits and folder structure matter.
4. If something breaks, **explain what you would check** — debugging out loud is evidence.

---

## After Showcase

Add your showcase reflection to `reflections/` and link any recording your teacher requests.
